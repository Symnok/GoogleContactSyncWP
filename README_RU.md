# GoogleContactSyncWP

Приложение для Windows Phone 8.1 Silverlight, которое синхронизирует контакты Google с хабом «Люди» на телефоне через Google People API и OAuth2 Device Flow. Поддерживает двунаправленную синхронизацию с разрешением конфликтов и фоновую автосинхронизацию.

## Возможности

- **Google → Телефон** — загружает все контакты из Google и записывает их в хаб «Люди» без UI-диалогов
- **Телефон → Google** — выгружает новые и изменённые контакты обратно в Google
- **Двунаправленное разрешение конфликтов** — если обе стороны изменились, сравниваются `updateTime` Google и локальный хэш; по умолчанию побеждает Google
- **Инкрементальная синхронизация по ETag** — при повторных синхронизациях загружаются только изменённые контакты
- **Обнаружение изменений по локальному хэшу** — в Google выгружаются только реально изменённые контакты
- **Фоновая автосинхронизация** — настраиваемый интервал: 30 / 60 / 120 / 240 минут
- **OAuth2 Device Flow** — авторизация через ввод кода в браузере на любом устройстве; redirect URI не требуется
- **Загрузка учётных данных с SD-карты** — сканирует `D:\` и подпапки в поисках `client_secret.json`
- **Живой лог синхронизации** — прогресс в реальном времени на отдельной вкладке «Log» с возможностью отправки по email
- **Обновление плитки** — после фоновой синхронизации на обратной стороне плитки отображается количество изменений

## Требования

### Устройство
- Windows Phone 8.1 (Silverlight runtime)
- Протестировано на Nokia Lumia 640
- Рекомендуется телефон с interop unlock (для доступа к файлам SD-карты)

### Разработка
- Visual Studio 2015
- Windows Phone 8.1 SDK
- Два проекта в решении: `GoogleContactSyncWP` (основное приложение) + `SyncAgent` (фоновая задача)

### Аккаунт Google
- Аккаунт Google с включённым Contacts API
- OAuth2-учётные данные (тип «Приложение для рабочего стола») из Google Cloud Console
- Скачанный файл `client_secret.json`

## Настройка

### Шаг 1 — Google Cloud Console

1. Перейдите на https://console.cloud.google.com
2. Создайте проект или выберите существующий
3. Включите **People API**
4. Перейдите в **Credentials → Create Credentials → OAuth Client ID**
5. Тип приложения: **Desktop app** (или TV/Limited Input Devices)
6. Скачайте файл `client_secret.json`

### Шаг 2 — Разместите учётные данные на телефоне

**Вариант А — SD-карта** (телефон с interop unlock):
- Скопируйте `client_secret.json` в корень SD-карты (`D:\` на телефоне)
- В приложении нажмите **Load client_secret.json** — будут просканированы `D:\` и `D:\Documents\`

**Вариант Б — Isolated Storage Explorer**:
- Подключите телефон по USB
- Откройте **Windows Phone Power Tools** (бесплатно)
- Перейдите в хранилище приложения `GoogleContactSyncWP`
- Перетащите `client_secret.json` в корень хранилища
- В приложении нажмите **Load client_secret.json**

**Вариант В — Ввод вручную**:
- Введите Client ID и Client Secret напрямую в текстовые поля на вкладке «Sign in»

### Шаг 3 — Вход в аккаунт

1. Откройте приложение → вкладка **Sign in**
2. Нажмите **Load client_secret.json** или введите данные вручную
3. Нажмите **Sign in with Google**
4. На компьютере откройте указанный URL и введите отображаемый код
5. Приложение автоматически опрашивает сервер — после авторизации переключается на вкладку Sync

### Шаг 4 — Синхронизация

- **Google → Phone** — первый запуск загружает все контакты; последующие — только изменения
- **Phone → Google** — выгружает новые и изменённые контакты телефона в Google
- **Sign out** — очищает токен и состояние синхронизации

## Фоновая синхронизация

1. Перейдите на вкладку **Settings**
2. Выберите интервал: 30 / 60 / 120 / 240 минут
3. Нажмите **Enable auto sync**
4. После каждой фоновой синхронизации обратная сторона плитки обновляется с количеством изменений

> **Примечание:** WP8.1 устанавливает минимальный интервал 30 минут для фоновых задач независимо от настроек. Реальная частота запуска зависит от режима экономии батареи и состояния сети.

## Структура проекта

```
GoogleContactSyncWP/
├── Models/
│   └── GoogleContact.cs          — модель данных контакта
├── Helpers/
│   ├── CredentialStorage.cs      — хранение refresh-токена и учётных данных в IsolatedStorageSettings
│   ├── ETagStorage.cs            — устаревшее хранилище ETag (заменено SyncStateStorage)
│   └── SyncStateStorage.cs       — состояние синхронизации: ETag + updateTime + локальный хэш
├── Services/
│   ├── GoogleApiService.cs       — Device Flow, People API: fetch/create/update через HttpWebRequest
│   ├── JsonHelper.cs             — ручной парсер и построитель JSON (без внешних зависимостей)
│   ├── ContactStoreService.cs    — чтение/запись контактов через Windows.Phone.PersonalInformation
│   └── SyncManager.cs            — логика двунаправленной синхронизации с разрешением конфликтов
├── MainPage.xaml / .cs           — интерфейс с 4 вкладками: Sign in, Sync, Settings, Log
├── App.xaml / .cs                — жизненный цикл приложения + обработчик сбоев
└── Properties/
    └── WMAppManifest.xml         — возможности приложения, объявление фоновой задачи

SyncAgent/
└── ScheduledAgent.cs             — самодостаточная фоновая задача,
                                    выполняет логику SyncManager каждые N минут
```

## Как работает синхронизация

### Google → Телефон

```
1. Загрузить состояние синхронизации (ETag + updateTime + localHash для каждого контакта)
2. Получить все контакты из Google People API
3. Для каждого контакта Google:
   - Нет на телефоне → скачать
   - ETag изменился → проверить, изменился ли локальный контакт
     - Изменился только Google → скачать
     - Изменились оба → сравнить updateTime → побеждает Google (по умолчанию)
4. Контакты, удалённые из Google → удалить локально
5. Сохранить новое состояние синхронизации
```

### Телефон → Google

```
1. Загрузить состояние синхронизации
2. Прочитать все контакты из ContactStore
3. Для каждого контакта телефона:
   - Нет RemoteId → создать в Google
   - LocalHash не изменился → пропустить
   - LocalHash изменился → обновить в Google
4. Сохранить обновлённые хэши
```

## Соответствие полей контакта

| Google People API | Хаб «Люди» Windows Phone |
|-------------------|--------------------------|
| `givenName` | Имя |
| `familyName` | Фамилия |
| `nickname` | Псевдоним |
| `biographies` | Заметки |
| `phoneNumbers` (mobile) | Мобильный телефон |
| `phoneNumbers` (home) | Домашний телефон |
| `phoneNumbers` (work) | Рабочий телефон |
| `phoneNumbers` (alternate) | Дополнительный моб./дом./раб. |
| `emailAddresses` (home) | Email |
| `emailAddresses` (work) | Рабочий email |
| `emailAddresses` (other) | Другой email |
| `organizations.name` | Компания |
| `organizations.title` | Должность |
| `urls` | Веб-сайт |
| `birthdays` | День рождения |

Телефоны без метки по умолчанию считаются **мобильными**. Email без метки по умолчанию считается **домашним**.

## Сборка

1. Откройте `GoogleContactSyncWP.sln` в VS2015
2. Установите конфигурацию **Release / ARM**
3. **Build → Build Solution**
4. XAP-файл: `GoogleContactSyncWP\Bin\ARM\Release\GoogleContactSyncWP_Release_ARM.xap`
5. Установка через USB: **Tools → Windows Phone → Application Deployment**

## Ограничения

- Распространение только через sideload — Store не поддерживается
- Минимальный интервал фоновой синхронизации: 30 минут (ограничение ОС)
- Фото контактов не синхронизируются
- Адреса не синхронизируются (ограничение API Windows.Phone.PersonalInformation)
